# 🚀 Запуск автоматической миграции

## Что делает скрипт:

1. ✅ Подключается к Amvera PostgreSQL
2. ✅ Создаёт все таблицы (users, categories, profiles, products, orders, order_items, messages, magic_links)
3. ✅ Импортирует данные из CSV файлов
4. ✅ Проверяет количество записей
5. ✅ Выводит отчёт о миграции

---

## 📋 Подготовка

### 1. Убедись, что `.env.local` создан

Файл должен содержать:
```env
DATABASE_URL=postgresql://threep_user:64p-a5R-vuT-8ts@threep-db-threep.db-msk0.amvera.tech:5432/threep
```

Если файла нет — открой `CREATE_ENV_LOCAL.txt` и создай `.env.local` по инструкции.

### 2. Убедись, что CSV файлы на месте

Проверь папку `Supabase CSV/`:
- ✅ `users.csv`
- ✅ `categories_rows.csv`
- ✅ `profiles_rows.csv`
- ✅ `products_rows.csv`
- ✅ `orders_rows.csv`
- ✅ `order_items_rows.csv`
- ✅ `messages_rows.csv`

### 3. Установи зависимости (если ещё не установлены)

```bash
npm install
```

---

## 🎯 Запуск миграции

### Вариант 1: Через npm скрипт (РЕКОМЕНДУЮ)

```bash
npm run migrate
```

### Вариант 2: Напрямую через Node.js

```bash
node scripts/migrate-to-amvera.js
```

## ⚠️ Возможные ошибки

### Ошибка: "relation already exists"

**Причина:** Таблицы уже созданы в БД.

**Решение:** Удали все таблицы и запусти скрипт заново, или создай скрипт очистки:

```bash
node scripts/clean-database.js
```

(Я могу создать этот скрипт, если нужно)

---

### Ошибка: "CSV файл не найден"

**Причина:** CSV файлы не в папке `Supabase CSV/`.

**Решение:** Проверь, что все CSV файлы экспортированы и находятся в правильной папке.

---

### Ошибка: "connection refused"

**Причина:** Неверный `DATABASE_URL` или БД недоступна.

**Решение:** 
1. Проверь `.env.local`
2. Проверь, что БД запущена на Amvera
3. Проверь, что IP-адрес не заблокирован
