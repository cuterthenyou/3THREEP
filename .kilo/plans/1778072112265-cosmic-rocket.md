# План переноса сайта threep с Vercel на российский хостинг

## Контекст проекта

**Стек:** Next.js 16 + React 18 + TypeScript + Tailwind CSS  
**База данных:** Supabase (PostgreSQL) — внешний сервис  
**Аутентификация:** Resend.com (email magic links)  
**Текущий хостинг:** Vercel (заблокирован в РФ через Cloudflare CDN)

---

## Сравнение вариантов хостинга

### 1. Amvera Cloud (amvera.ru)

**Тип:** PaaS (Platform as a Service), аналог Heroku/Vercel  
**Дата-центры:** Москва (RU) + Варшава (EU)  
**Работает в РФ без VPN:** ДА

**Тарифы для Next.js приложения:**
| Тариф | Цена | RAM | CPU | Хранилище |
|-------|------|-----|-----|-----------|
| Начальный Плюс | 490 ₽/мес | 1 ГБ | 0.5 vCPU | 7 ГБ |
| Стандартный | 1450 ₽/мес | 2.5 ГБ | 1 vCPU | 15 ГБ |
| Ультра | 4300 ₽/мес | 6 ГБ | 2 vCPU | 25 ГБ |

**Деплой Next.js:** Поддерживается через Node.JS Server окружение. Нужен файл `amvera.yaml`:
```yaml
meta:
  environment: node
  toolchain:
    name: npm
    version: 20

build:
  additionalCommands: npm run build

run:
  command: npm run start
  containerPort: 3000
```

**Плюсы:**
- Самый простой деплой — git push, как Vercel
- Российская компания, серверы в Москве
- Бесплатный SSL, привязка домена
- 111 ₽ на старте (тест)
- Managed PostgreSQL от 290 ₽/мес (если нужно мигрировать с Supabase)
- Нет необходимости в Dockerfile или Kubernetes
- Авто-перезапуск при сбоях, бэкапы

**Минусы:**
- Маленькая компания (риск закрытия)
- Нет готового примера с Next.js в документации
- На тарифе "Начальный Плюс" (490 ₽) может не хватать памяти для Next.js build (нужен минимум 1 ГБ)
- Supabase остаётся внешним (не блокируется в РФ по API — это нормально)

**Рекомендуемый тариф для старта:** Стандартный (1450 ₽/мес) — 2.5 ГБ RAM достаточно для Next.js

### 2. Yandex Cloud

**Тип:** IaaS/PaaS, полный облачный провайдер  
**Дата-центры:** Москва, Санкт-Петербург  
**Работает в РФ без VPN:** ДА

**Варианты для Next.js:**

**Вариант A — Виртуальная машина (Compute Cloud):**
- Минимум для Next.js: 2 vCPU + 4 ГБ RAM
- Стоимость: ~2300 ₽/мес
- Требует самостоятельной настройки: Nginx, PM2, SSL, CI/CD

**Вариант B — Serverless Containers:**
- Платишь только за вызовы (от 0 до нескольких сотен рублей)
- Нужен Docker-образ
- Холодный старт может быть заметен пользователям
- Сложнее настроить Next.js SSR

**Вариант C — Cloud Functions:**
- Не подходит для Next.js напрямую (нет стандартного адаптера)

**Плюсы:**
- Крупная компания, высокая надёжность
- Managed PostgreSQL (можно заменить Supabase)
- Managed Kubernetes для масштабирования
- CDN работает в России (собственный, не Cloudflare)

**Минусы:**
- Сложная настройка (нужно знать DevOps)
- Нет аналога "git push деплой" из коробки
- Дороже при базовой конфигурации
- Нужно настраивать CI/CD самостоятельно (GitHub Actions → Yandex Cloud)

### 3. Selectel

**Тип:** IaaS/PaaS, российский облачный провайдер  
**Дата-центры:** Санкт-Петербург, Москва  
**Работает в РФ без VPN:** ДА

**Тарифы (облачные серверы):**
| Конфигурация | Цена |
|---|---|
| 1 vCPU + 1 ГБ RAM | ~957 ₽/мес |
| 2 vCPU + 4 ГБ RAM | ~2308 ₽/мес |
| 2 vCPU + 8 ГБ RAM | ~3281 ₽/мес |

**Плюсы:**
- Крупная российская компания, надёжная
- Managed Kubernetes (для масштабирования)
- Managed БД (PostgreSQL, MySQL, ClickHouse)
- CDN работает в РФ
- Container Registry (Docker образы)

**Минусы:**
- Нет простого "git push деплой" как в Vercel/Amvera
- Нужно самостоятельно настраивать: Docker, Nginx, CI/CD
- Дороже Amvera для простого Next.js приложения

## Рекомендация

**Для вашего проекта рекомендую Amvera Cloud** по следующим причинам:

1. **Наиболее похоже на Vercel** — деплой через `git push`, тот же опыт разработчика
2. **Самый дешёвый старт** — 1450 ₽/мес для полноценного Next.js
3. **Supabase остаётся** — API Supabase работает из России через их серверы в EU, не через Cloudflare CDN (это разные вещи)
4. **Resend.com** — тоже API-сервис, доступен из РФ через программный запрос с сервера Amvera
5. **Нет необходимости в DevOps** — не нужны Nginx, Docker, Kubernetes

**Если нужна максимальная надёжность** (для продакшена с серьёзной нагрузкой) — Selectel или Yandex Cloud с виртуальной машиной, но это требует настройки DevOps.

## Что нужно будет сделать для деплоя на Amvera

### Шаг 1: Создать `amvera.yaml` в корне проекта

```yaml
meta:
  environment: node
  toolchain:
    name: npm
    version: 20

build:
  additionalCommands: npm run build

run:
  command: npm run start
  containerPort: 3000
```

### Шаг 2: Убедиться что `next.config.ts` настроен для production

В `package.json` скрипт `start` должен быть:
```json
"start": "next start -p 3000"
```

### Шаг 3: Зарегистрироваться на amvera.ru

- Создать проект типа "Node.JS Server"
- Выбрать тариф "Стандартный" (1450 ₽/мес)
- Привязать собственный домен

### Шаг 4: Настроить переменные окружения

Перенести все переменные из `.env.local` в панель управления Amvera:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`
- Переменные Resend
- Все остальные секреты

### Шаг 5: Деплой

```bash
git remote add amvera <amvera-git-url>
git push amvera master
```

### Шаг 6: Проверить доступность Supabase и Resend из РФ

- Supabase API (`*.supabase.co`) — работает из РФ (не заблокировано)
- Resend API (`api.resend.com`) — работает из РФ с серверов Amvera

## Риски

1. **Amvera — небольшая компания**: Если закроется, нужна быстрая миграция. Риск умеренный, компания активно развивается.
2. **Supabase может заблокировать**: Если Supabase заблокируют в РФ — нужно будет мигрировать БД на Yandex Managed PostgreSQL или Amvera PostgreSQL. Это отдельная задача.
3. **Resend может перестать работать**: Альтернативы — Yandex Cloud Postbox (российский аналог), SMTP через Mail.ru.

---

## Решение принято

- **Хостинг:** Amvera Cloud
- **База данных:** Supabase оставить (API работает из РФ без VPN)
- **Email:** Resend оставить (API работает с серверов Amvera)

## План реализации (задачи)

### Задача 1: Создать `amvera.yaml` в корне проекта
Файл конфигурации для деплоя Next.js на Amvera:
```yaml
meta:
  environment: node
  toolchain:
    name: npm
    version: 20

build:
  additionalCommands: npm run build

run:
  command: npm run start
  containerPort: 3000
```

### Задача 2: Проверить и обновить `package.json`
Убедиться что скрипт `start` запускает Next.js на порту 3000:
```json
"start": "next start -p 3000"
```

### Задача 3: Проверить `next.config.ts`
Убедиться что нет жёстких привязок к Vercel-специфичным функциям.

### Задача 4: Зарегистрироваться на amvera.ru и создать проект
- Зарегистрироваться на https://cloud.amvera.ru
- Создать новый проект типа "Приложение"
- Выбрать тариф "Стандартный" (1450 ₽/мес, 2.5 ГБ RAM)
- Получить git URL вида `https://git.amvera.ru/<username>/<project>`

### Задача 5: Перенести переменные окружения
В панели Amvera добавить все переменные из `.env.local`:
- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`  
- `SUPABASE_SERVICE_ROLE_KEY`
- Переменные Resend API
- Все остальные секреты

### Задача 6: Первый деплой
```bash
git remote add amvera https://git.amvera.ru/<username>/<project>
git push amvera master
```

### Задача 7: Привязать домен
В панели Amvera: настройки проекта → Домены → добавить собственный домен.
Обновить DNS-записи у регистратора домена.

### Задача 8: Удалить деплой с Vercel (по желанию)
После проверки что сайт работает на Amvera — можно отключить Vercel.
